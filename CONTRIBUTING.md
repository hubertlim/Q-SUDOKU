# Contributing

Thanks for your interest in Q-SUDOKU.

## Getting started

1. Fork the repo
2. Run locally: `docker compose up --build`
3. Open `http://localhost:8080`
4. Make your changes — the app is plain HTML/CSS/JS, no build step

## Guidelines

- Keep it minimal — no frameworks, no bundlers
- Test in both desktop and mobile viewports
- Respect the existing code style

## Submitting

Open a PR against `main` with a clear description of what you changed and why.
